# 层级看板视觉设计对比

## 设计演变

### 优化前：传统树形连接线
```
TagGroupManager
│
├── 实现
│   ├── 标签解析
│   ├── 多级展开
│   └── UI
│       └── 折叠逻辑
│
└── 需求
    ├── 历史遗留
    └── 用户反馈
```

**问题：**
- ❌ 连接线多而杂乱
- ❌ 层级深时难以追踪
- ❌ 视觉噪音大
- ❌ 看起来过时

---

### 优化后：现代简洁设计
```
TagGroupManager

  ● 实现
    ● 标签解析
    ● 多级展开
    ● UI
      ● 折叠逻辑

  ● 需求
    ● 历史遗留
    ● 用户反馈
```

**优势：**
- ✅ 简洁的圆点标记
- ✅ 渐变边框引导视线
- ✅ 清晰的缩进层级
- ✅ 现代化设计

---

## 视觉元素详解

### 1. 圆点标记（Dot Indicator）

**静态状态：**
- 大小：4px × 4px
- 颜色：`var(--text-muted)`
- 透明度：40%
- 位置：节点左侧 8px

**悬停状态：**
- 大小：6px × 6px
- 颜色：`var(--interactive-accent)`
- 透明度：100%
- 效果：发光（box-shadow）

**作用：**
- 标记节点位置
- 提供视觉锚点
- 增强可点击性

---

### 2. 渐变边框（Gradient Border）

**样式：**
```css
background: linear-gradient(
  to bottom,
  主题色 0%,
  透明 100%
);
```

**静态状态：**
- 宽度：2px
- 透明度：30%
- 位置：子节点容器左侧

**悬停状态：**
- 宽度：3px
- 透明度：60%

**作用：**
- 表示父子关系
- 引导视线向下
- 不会显得杂乱

---

### 3. 渐进式缩进（Progressive Indent）

**层级缩进：**
- 第 1 层：16px（margin-left）+ 12px（padding-left）
- 第 2 层：再增加 28px
- 第 3 层：再增加 28px
- 以此类推...

**深度表现：**
- 第 0 层（根）：font-weight: 600, font-size: 0.95em
- 第 1 层：opacity: 0.95
- 第 2 层：opacity: 0.90
- 第 3 层：opacity: 0.85

**作用：**
- 清晰的层级结构
- 视觉深度感
- 易于扫描

---

### 4. Pill 样式（Node Pill）

**基础样式：**
- 圆角：12px
- 内边距：6px 12px
- 背景：`var(--background-secondary-alt)`
- 边框：1px transparent

**悬停效果：**
- 背景：`var(--interactive-accent)`
- 颜色：`var(--text-on-accent)`
- 变换：`translateX(4px) scale(1.02)`
- 阴影：`0 2px 8px rgba(0, 0, 0, 0.15)`

**彩色标签：**
- 左边框：3px solid（标签颜色）
- 悬停时：4px solid

---

## 交互动画

### 悬停序列

**1. 鼠标移入节点区域**
```
圆点：4px → 6px，灰色 → 主题色，添加发光
Pill：向右移动 4px，轻微放大
边框：2px → 3px，透明度 30% → 60%
```

**2. 鼠标移出节点区域**
```
所有元素平滑恢复到初始状态
过渡时间：0.2s ease
```

---

## 颜色系统

### 主题适配

**浅色主题：**
- 背景：`rgba(255, 255, 255, 0.75)`
- 边框：`rgba(0, 0, 0, 0.05)`
- 圆点：较深的灰色

**深色主题：**
- 背景：`rgba(30, 30, 30, 0.65)`
- 边框：`rgba(255, 255, 255, 0.08)`
- 圆点：较浅的灰色

**主题色：**
- 使用 `--interactive-accent`
- 自动适配用户的主题设置
- 保持一致的视觉语言

---

## 空间布局

### 垂直间距
```
树容器：padding: 4px 0
节点间距：margin-top: 2px
子容器：margin-top: 4px
内容：padding: 4px 0
```

### 水平间距
```
子容器左边距：margin-left: 16px
子容器左内边距：padding-left: 12px
圆点位置：left: -8px
内容左内边距：padding-left: 4px
```

```

### 过渡优化

```

### 选择器效率
```css
.tgm-hb-item[data-depth="0"] > .tgm-hb-content-wrapper > .tgm-hb-pill
/* 使用属性选择器和直接子选择器 */
```

## 可访问性

### 对比度
- 圆点：静态时 40% 透明度，悬停时 100%
- Pill：使用主题的标准颜色变量
- 边框：渐变确保不会过于突出

### 交互区域
- Pill 最小尺寸：约 24px 高度
- 悬停时变大，更易点击
- 圆点悬停时从 4px 变为 6px

### 视觉反馈
- 悬停时多重反馈（颜色、大小、位置）
- 动画流畅（0.2s ease）
- 清晰的状态变化

---

## 设计原则

### 1. 简洁至上
> "Perfection is achieved not when there is nothing more to add, but when there is nothing left to take away."

- 去除不必要的连接线
- 用最少的元素表达最多的信息
- 让内容成为焦点

### 2. 渐进增强
- 静态时保持简洁
- 交互时提供丰富反馈
- 不同层级有不同的视觉权重

### 3. 一致性
- 遵循 Obsidian 设计语言
- 使用主题变量
- 与其他 UI 元素协调

### 4. 可发现性
- 悬停效果明显
- 圆点发光引导注意力
- Pill 移动暗示可点击

## 实现细节

### 关键 CSS 技巧

**1. 渐变边框**
```css
.tgm-hb-children::before {
  content: '';
  position: absolute;
  left: 0;
  top: 0;
  bottom: 0;
  width: 2px;
  background: linear-gradient(
    to bottom,
    var(--interactive-accent-hover) 0%,
    transparent 100%
  );
  opacity: 0.3;
  border-radius: 2px;
}
```

**2. 圆点标记**
```css
.tgm-hb-content-wrapper::before {
  content: '';
  position: absolute;
  left: -8px;
  top: 50%;
  transform: translateY(-50%);
  width: 4px;
  height: 4px;
  background: var(--text-muted);
  border-radius: 50%;
  opacity: 0.4;
  transition: all 0.2s;
}
```

**3. 悬停动画**
```css
.tgm-hb-pill:hover {
  background: var(--interactive-accent);
  color: var(--text-on-accent);
  transform: translateX(4px) scale(1.02);
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.15);
  border-color: var(--interactive-accent);
}
```

## 总结

这次优化通过以下方式改善了层级看板的视觉体验：

1. **简化连接线** → 渐变边框 + 圆点标记
2. **增强层级感** → 缩进 + 透明度 + 字重
3. **丰富交互** → 多重悬停效果
4. **现代设计** → 圆角、阴影、动画

结果是一个更简洁、更美观、更易用的层级看板界面。
