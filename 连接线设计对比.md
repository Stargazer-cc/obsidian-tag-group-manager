# 连接线设计对比

## 设计演变

### 方案 1：只有垂直线 + 圆点
```
TagGroupManager
│
● 实现
  ● 标签解析
  ● UI
│
● 需求
```
**问题：** 无法清晰看出节点和垂直线的关系

---

### 方案 2：垂直线 + 水平线 + 圆点 ✅
```
TagGroupManager
│
├─ ● 实现
│  ├─ ● 标签解析
│  └─ ● UI
│
└─ ● 需求
```
**优势：** 清晰的树形结构，层级关系一目了然

---

## 视觉元素说明

### 垂直连接线 `│`
- **作用**：表示"这是一组子节点"
- **样式**：1px 灰色实线
- **位置**：子节点容器左侧

### 水平连接线 `├─` 或 `└─`
- **作用**：表示"从父节点分支到子节点"
- **样式**：1px 灰色实线，8px 长
- **位置**：从垂直线到圆点

### 圆点标记 `●`
- **作用**：标记节点位置
- **样式**：4px 圆点
- **位置**：水平线末端

## 设计优势

✅ **清晰的层级关系**
- 垂直线表示父子关系
- 水平线表示分支关系
- 圆点标记节点位置

✅ **符合视觉习惯**
- 类似文件管理器
- 类似命令行 tree 命令
- 用户一眼就懂

✅ **简洁美观**
- 只使用灰色系
- 线条细腻（1px）
- 圆点小巧（4px）

✅ **交互友好**
- 悬停时连接线变深
- 圆点发光提供反馈
- 清晰表明当前节点
