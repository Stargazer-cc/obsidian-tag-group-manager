# Live Preview标签颜色功能测试报告

## 测试日期
2026-03-20

## 功能描述
实现了标签颜色在Live Preview（实时预览/编辑模式）中的显示支持，并模仿了Obsidian笔记属性中的标签样式。之前标签颜色只能在阅读模式下显示，现在在编辑模式下也能正常显示，且样式与属性标签保持一致。

### 属性标签样式分析
通过检查Obsidian属性标签的实际样式，提取了以下特征：
- 背景色：rgba(r, g, b, 0.2) - 20%透明度
- 文字色：rgb(r*0.65, g*0.65, b*0.65) - 原色的65%亮度
- 无边框：border: none
- 圆角：13px（较大的圆角，呈现胶囊状）
- 内边距：2px 8px（上下2px，左右8px）

- 高度：17px
- 显示方式：inline-flex，居中对齐

### 应用的样式

- 背景色：rgba(r, g, b, 0.2)
- 文字色：rgb(r*0.65, g*0.65, b*0.65)

## 技术实现

### 问题分析
CodeMirror 6（Obsidian使用的编辑器）不会自动为标签添加特定的类名。标签元素只有通用的类名：
- `.cm-hashtag-begin` - #号部分
- `.cm-hashtag-end` - 标签文本部分
- `.cm-tag-` - 通用标签类（不包含标签名）

### 解决方案
1. **使用MutationObserver监听DOM变化**
   - 监听整个文档的DOM变化
   - 当检测到新的标签元素时，自动处理

2. **动态添加data属性**
   - 为有颜色配置的标签添加`data-tag-name`属性
   - 属性值为标签名（特殊字符替换为连字符）
   - 同时为#号和标签文本部分添加属性

3. **CSS选择器更新**
   - 使用`[data-tag-name="标签名"]`属性选择器
   - 为#号和标签文本分别设置圆角和内边距
   - 保持与属性标签一致的视觉效果

## 代码修改

### 新增属性
```typescript
private livePreviewObserver: MutationObserver | null = null;
```

### 新增方法
1. `startLivePreviewObserver()` - 启动DOM监听器
2. `stopLivePreviewObserver()` - 停止DOM监听器
3. `processExistingTags()` - 处理现有标签
4. `processTagElement(element)` - 处理单个标签元素

### 修改方法
1. `applyTextTagStyling()` - 更新CSS生成逻辑，模仿属性标签样式，启动监听器
2. `onunload()` - 添加监听器清理代码
