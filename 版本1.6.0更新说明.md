# Tag Group Manager v1.6.0 更新说明

## 发布日期
2026-03-20

## 版本概述
v1.6.0是一个重大功能更新版本，主要包含多级标签展开功能的全面升级、标签颜色样式的完全重构以及自动添加标签功能的优化。本次更新大幅提升了插件对多级标签的支持能力，并使标签样式与Obsidian原生属性标签保持一致。

---

## ✨ 核心功能增强

### 🏷️ 多级标签展开功能全面升级

#### 1. 灵活的展开深度选择
支持2级和3级两种展开模式，用户可根据标签组织需求自由选择：

**3级展开模式（默认）：**
- 结构：`A/B/C/D/E` → 组集=A，标签组=B，标签=C、C/D、C/D/E
- 适用场景：需要组集来组织大量标签组的情况
- 示例：
  ```
  #前端/框架/React/Hooks/useState
  → 组集"前端"
    → 标签组"框架"
      → 标签"React"、"React/Hooks"、"React/Hooks/useState"
  ```

**2级展开模式：**
- 结构：`A/B/C/D` → 标签组=A，标签=B、B/C、B/C/D
- 适用场景：标签分类较少，不需要组集的情况
- 示例：
  ```
  #框架/React/Hooks/useState
  → 标签组"框架"
    → 标签"React"、"React/Hooks"、"React/Hooks/useState"
  ```

#### 2. 选择性展开
新增"指定展开的第一级标签"输入框，支持过滤展开：
- 输入要展开的第一级标签名称，多个标签用逗号分隔
- 例如：`前端,后端,数据库`
- 留空则展开所有多级标签
- 只有匹配的第一级标签会被展开到组集和标签组

#### 3. 智能节点收录
- **完整收录**：自动收录所有末节点（叶子节点），包括深度>3的标签
- **旧版问题**：之前深度>3的叶子节点会被遗漏
- **新版解决**：所有节点都会被添加到对应的标签组中
- **示例**：`前端/框架/React/Hooks/useState`中的`useState`现在会被正确收录

#### 4. 两级标签特殊处理
在3级展开模式下，对于只有两级的多级标签（如`前端/React`）：
- 直接以第一级为名生成标签组
- 不创建组集
- 标签组中包含该两级标签
- 避免为少量两级标签创建不必要的组集

#### 5. 防命名冲突
**组集名重复：**
- 如果组集名已存在，自动添加序号：`前端 (1)`、`前端 (2)`
- 确保不会覆盖已有的组集

**标签组名重复：**
- 如果标签组名已存在，自动添加序号：`框架 (1)`、`框架 (2)`
- 确保不会覆盖已有的标签组

**安全重复展开：**
- 可以安全地多次运行展开功能
- 不会破坏已有的配置
- 新标签会被追加到末尾

#### 6. 智能显示策略
根据不同界面的需求，采用不同的显示策略：

**设置页面：**
- 显示所有标签，包括所有深度的末节点
- 用户可以为所有标签设置颜色
- 完全透明，用户可见所有内容

**标签选择器和标签组视图：**
- 隐藏深度≥minDepth的自动展开叶子节点
- 显示所有其他标签（非多级、深度<minDepth、有子节点的父节点）
- 保持界面简洁，避免过多标签

**悬浮式层级看板：**
- 显示完整的树形结构，包括所有末节点
- 支持点击任意节点插入标签
- 支持颜色显示
- 提供完整的标签访问能力

#### 7. 尊重用户操作
**手动添加的标签：**
- 始终显示在所有界面
- 不会被隐藏
- 符合用户预期

**手动删除的标签：**
- 不会被重新添加
- 即使重新运行展开功能
- 尊重用户的删除决定

**自动展开的标签：**
- 会被记录到`autoExpandedTags`字段
- 在选择器和视图中可能被隐藏（根据深度）
- 在设置页面始终可见

### 🎨 标签颜色样式全面升级

#### 1. 模仿属性标签样式
正文标签样式完全模仿Obsidian笔记属性中的标签样式：

**视觉特征：**
- **胶囊状外观**：13px圆角，呈现柔和的胶囊形状
- **背景色**：rgba(r, g, b, 0.2) - 20%透明度，更加柔和
- **文字色**：rgb(r*0.65, g*0.65, b*0.65) - 原色的65%亮度，更加深色
- **无边框**：移除边框，更加简洁
- **内边距**：左右8px，更加宽松舒适
- **字体大小**：13px，与属性标签一致
- **高度**：17px，保持一致性

**对比旧版：**
| 特性 | 旧版 | 新版 |
|------|------|------|
| 背景透明度 | 15% | 20% |
| 文字颜色 | 纯色 | 65%亮度 |
| 边框 | 1px实线 | 无边框 |
| 圆角 | 4px | 13px |
| 内边距 | 2px | 8px |
| 外观 | 方形 | 胶囊状 |

#### 2. Live Preview支持
标签颜色现在在编辑模式（Live Preview）中也能正常显示：

**技术实现：**
- 使用MutationObserver动态监听DOM变化
- 当检测到新的标签元素时，自动处理
- 为有颜色配置的标签添加`data-tag-name`属性
- 使用CSS属性选择器应用样式

**特点：**
- 自动处理新添加的标签
- 与阅读模式保持完全一致的视觉效果
- 无性能问题，响应迅速
- 支持所有颜色（预设颜色和自定义颜色）

**兼容性：**
- ✅ 支持多级标签（包含/）
- ✅ 支持中文标签
- ✅ 支持特殊字符标签
- ✅ 与阅读模式样式完全一致

### 🔄 自动添加标签功能优化

#### 1. 完整节点收录
- **问题修复**：修复了深度>3的叶子节点被遗漏的问题
- **现在行为**：所有符合规则的标签都会被收录
- **示例**：
  ```
  规则：react → 测试-前端框架
  标签：#react/hooks/useState
  结果：会被添加到"测试-前端框架"标签组
  ```

#### 2. 标记支持
- **新增字段**：`autoExpandedTags`数组
- **用途**：区分自动添加和手动添加的标签
- **好处**：
  - 可以实现智能显示策略
  - 尊重用户的删除操作
  - 支持安全的重复扫描

#### 3. 尊重删除操作
- **智能检测**：检查标签是否被用户手动删除
- **行为**：
  - 如果标签在`autoExpandedTags`中但不在`tags`中，说明被用户删除
  - 不会重新添加被删除的标签
  - 新标签会正常添加

#### 4. 智能扫描
- **启动扫描**：支持在插件启动时自动扫描
- **规则匹配**：根据设置的规则自动添加标签
- **示例规则**：
  ```
  react:测试-前端框架
  vue:测试-前端框架
  python:测试-后端语言
  ```

## 🐛 问题修复

### 1. 输入框无法输入
- **问题**：展开配置中的输入框无法输入
- **原因**：宽度设置不当
- **解决**：设置为100%宽度

### 2. 深度>3的标签被遗漏
- **问题**：展开时深度>3的叶子节点不会被收录
- **原因**：展开逻辑只处理前3层
- **解决**：收录所有节点，从指定层级到最后一层

### 3. 重复展开影响已编辑标签组
- **问题**：重复展开会改变标签顺序、删除手动添加的标签
- **原因**：没有区分自动添加和手动添加
- **解决**：
  - 新增`autoExpandedTags`字段标记
  - 保持用户的标签顺序
  - 不删除手动添加的标签
  - 不重新添加被删除的标签

### 4. Live Preview模式下标签颜色不显示
- **问题**：标签颜色只在阅读模式显示
- **原因**：CodeMirror 6不会自动添加标签特定的类名
- **解决**：
  - 使用MutationObserver监听DOM变化
  - 动态添加`data-tag-name`属性
  - 使用CSS属性选择器应用样式

## 🔧 技术改进

### 1. 新增数据字段

**TagGroup接口：**
```typescript
interface TagGroup {
  id?: string;
  name: string;
  tags: string[];
  autoExpandedTags?: string[];  // 新增：记录自动添加的标签
}
```

**TagGroupManagerSettings接口：**
```typescript
interface TagGroupManagerSettings {
  // ...
  expandTagPrefixes: string;  // 新增：指定要展开的第一级标签
  expandDepth: number;        // 新增：展开深度（2或3）
}
```

### 2. 新增辅助函数

**shouldHideTag：**
- 判断标签是否应该被隐藏
- 考虑自动添加标记、深度、子节点等因素

**getGroupExpandDepth：**
- 判断标签组的展开深度
- 根据标签组是否在组集中判断

### 3. 新增观察器

**livePreviewObserver：**
- 类型：MutationObserver
- 用途：监听DOM变化，处理新添加的标签元素
- 生命周期：
  - 启动：`startLivePreviewObserver()`
  - 停止：`stopLivePreviewObserver()`
  - 清理：`onunload()`

### 4. 优化展开逻辑

**convertMultiLevelTagsToGroups：**
- 支持展开深度选择
- 支持前缀过滤
- 收录所有节点
- 防命名冲突
- 记录自动添加的标签

**autoScanTagsToGroups：**
- 完整节点收录
- 尊重删除操作
- 智能扫描

### 5. 优化渲染逻辑

**TagSelectorModal：**
- 根据展开深度过滤叶子节点
- 使用`shouldHideTag`判断

**TagGroupView：**
- 根据展开深度过滤叶子节点
- 使用`shouldHideTag`判断

**设置页面：**
- 显示所有标签（不过滤）
- 用户可以为所有标签设置颜色

### 6. 向后兼容性

**完全兼容旧数据：**
- `autoExpandedTags`是可选字段，旧数据没有也能正常工作
- `expandDepth`有默认值3，旧数据使用默认值
- 如果字段不存在或为空，所有标签被视为手动添加
- 用户升级插件后，旧数据继续正常工作
