## 建议的测试规则

以下是建议配置的测试规则（在插件设置中添加）：

1. `react:前端框架` - 测试关键字匹配
2. `^python.*:编程语言` - 测试正则表达式
3. `框架:技术栈` - 测试多级标签处理
4. `vue:前端框架` - 测试多规则匹配
5. `开发:开发类别` - 测试中文匹配

## 预期结果

### 规则：`react:前端框架`
应该匹配：
- react
- React
- react-native
- react-hooks
- 前端/React
- 前端/框架/React
- 前端/框架/React/Hooks
- 前端/框架/React/Hooks/useState（如果修复了问题1）
- 前端/框架/React/状态管理/Redux

### 规则：`^python.*:编程语言`
应该匹配：
- python
- python3
- python-django
- 编程语言/Python
- 编程语言/Python/Django

不应该匹配：
- cpython（不以python开头）

### 规则：`框架:技术栈`
应该匹配：
- 框架
- 前端框架
- 后端框架
- 测试框架
- 前端/框架/React
- 前端/框架/Vue
- 前端/框架/Angular
- 后端/框架/Express
- 后端/框架/Koa
- 测试/框架/Jest

## 测试步骤

1. 将此文件保存到Obsidian库中
2. 在插件设置中添加测试规则
3. 创建对应的标签组（如"前端框架"、"编程语言"、"技术栈"）
4. 点击"立即刷新"按钮
5. 检查标签组中是否添加了预期的标签
6. 验证是否存在报告中提到的问题

## 注意事项

- 测试前建议备份插件设置
- 可以通过删除标签组来重置测试
- 建议逐个规则测试，便于定位问题
