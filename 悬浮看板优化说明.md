# 悬浮式全层级看板优化说明

## 优化内容

### 1. 智能显示策略
- **多级标签**（包含 `/`）：显示悬浮式全层级看板，不显示 tooltip
- **非多级标签**：只显示 tooltip，不显示看板

### 2. 切换性能优化
- 立即取消待处理的隐藏操作，避免延迟
- 快速切换不同标签时，立即更新内容，不等待延迟
- 减少卡顿和显示上一个标签内容的问题

### 3. 优化位置
- ✅ TagSelectorModal（浮动标签选择器）
- ✅ TagGroupView（标签总览视图）
- ✅ 设置页面的标签组管理
- ✅ 批量筛选添加界面
- ✅ 标签库选择界面

## 测试场景

### 场景 1：快速切换多级标签
1. 打开浮动标签选择器或标签总览视图
2. 快速将鼠标在多个多级标签的 `>>` 按钮间移动
3. 观察看板是否能快速切换，不会卡顿或显示上一个标签

### 场景 2：多级标签 vs 非多级标签
1. 悬停在多级标签上（如 `项目/工作/重要`）
   - 应该显示悬浮看板
   - 不应该显示 tooltip
2. 悬停在非多级标签上（如 `重要`）
   - 应该显示 tooltip
   - 不应该显示悬浮看板

### 场景 3：点击固定看板
1. 点击多级标签的 `>>` 按钮
2. 看板应该固定显示
3. 需要手动点击关闭按钮才能关闭

## 技术细节

### HierarchyBoard.show() 优化
```typescript
// 立即取消任何待处理的隐藏操作
if (this.hTimer) {
    window.clearTimeout(this.hTimer);
    this.hTimer = null;
}

// 如果正在显示不同的标签，立即切换（不等待延迟）
this.currentRootTag = rootTag;
this.isVisible = true;
```

### 标签渲染逻辑优化
```typescript
const isMultiLevelTag = tag.includes('/');

// 多级标签：添加 hierarchy board trigger
if (isMultiLevelTag && this.plugin) {
    const triggerEl = tagEl.createDiv('tgm-hierarchy-trigger');
    // ... 添加事件监听器
}

// 非多级标签：只添加 tooltip
if (!isMultiLevelTag) {
    tagTextEl.setAttribute('aria-label', tag);
}
```
