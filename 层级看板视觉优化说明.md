# 层级看板视觉优化说明

## 优化目标
- ✅ 减少视觉噪音，去除复杂的连接线
- ✅ 保持清晰的层级关系
- ✅ 增强交互反馈
- ✅ 更现代、简洁的设计

## 设计方案

### 1. 渐进式缩进 + 渐变边框
**替代传统的树形连接线**

```
原来：使用 border-left 实线
现在：使用渐变边框（从主题色到透明）
```

**优势：**
- 视觉更柔和，不会显得杂乱
- 渐变效果增强了层级的深度感
- 悬停时边框变粗变亮，增强交互反馈

### 2. 圆点标记系统
**替代 L 形连接线**

```
每个节点前添加小圆点（4px）
悬停时圆点变大（6px）并发光
根节点不显示圆点
```

**优势：**
- 简洁明了，不占用太多空间
- 悬停时的动画效果增强了可点击性
- 圆点的发光效果提供了视觉焦点

### 3. 深度感表现
**通过多种视觉元素表现层级**

- **缩进**：每层增加 16px + 12px 的缩进
- **透明度**：深层节点略微降低透明度（0.95 → 0.85）
- **字重**：根节点使用粗体（font-weight: 600）
- **字号**：根节点略大（0.95em）

### 4. 增强的悬停效果

**Pill 悬停：**
- 向右移动 4px
- 轻微放大（scale: 1.02）
- 添加阴影
- 背景变为主题色

**连接线悬停：**
- 边框变粗（2px → 3px）
- 透明度增加（0.3 → 0.6）

**圆点悬停：**
- 变大（4px → 6px）
- 变为主题色
- 添加发光效果

## CSS 关键特性

### 渐变边框
```css
.tgm-hb-children::before {
  background: linear-gradient(
    to bottom,
    var(--interactive-accent-hover) 0%,
    transparent 100%
  );
  opacity: 0.3;
}
```

### 圆点标记
```css
.tgm-hb-content-wrapper::before {
  width: 4px;
  height: 4px;
  background: var(--text-muted);
  border-radius: 50%;
  opacity: 0.4;
}
```

### 悬停动画
```css
.tgm-hb-pill:hover {
  transform: translateX(4px) scale(1.02);
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.15);
}
```

## 视觉对比

### 优化前
```
├── 节点1
│   ├── 子节点1
│   │   ├── 孙节点1
│   │   └── 孙节点2
│   └── 子节点2
└── 节点2
```
- 多条实线连接
- 视觉杂乱
- 层级较多时难以追踪

### 优化后
```
  ● 节点1
    ● 子节点1
      ● 孙节点1
      ● 孙节点2
    ● 子节点2
  ● 节点2
```
- 简洁的圆点标记
- 渐变边框引导视线
- 清晰的缩进层级

## 测试场景

### 场景 1：浅层级（2-3 层）
- 圆点标记清晰可见
- 渐变边框不会显得多余
- 整体视觉简洁

### 场景 2：深层级（4-5 层）
- 渐变边框引导视线
- 透明度变化表现深度
- 不会出现连接线交织的情况

### 场景 3：悬停交互
- 圆点发光效果明显
- Pill 移动和放大流畅
- 边框变粗增强连接感

## 设计理念

### 1. 少即是多
- 去除不必要的视觉元素
- 用简单的圆点替代复杂的连接线
- 让内容本身成为焦点

### 2. 渐进增强
- 静态时保持简洁
- 交互时提供丰富的视觉反馈
- 悬停效果增强可发现性

### 3. 一致性
- 使用主题变量（`--interactive-accent`）
- 遵循 Obsidian 的设计语言
- 深色/浅色主题自动适配

## 技术细节

### 性能优化
- 使用 CSS transform 而非 position 变化
- 使用 opacity 而非 display 切换
- 所有动画使用 GPU 加速

### 可访问性
- 保持足够的对比度
- 悬停区域足够大（pill padding: 6px 12px）
- 圆点在悬停时变大，便于识别

### 响应式
- 使用相对单位（em, px）
- 自动适应不同的字体大小
- 在小屏幕上也能保持清晰

## 预期效果

✅ 视觉更简洁，不会显得杂乱
✅ 层级关系依然清晰
✅ 交互反馈更丰富
✅ 现代化的设计风格
✅ 更好的可读性和可用性
