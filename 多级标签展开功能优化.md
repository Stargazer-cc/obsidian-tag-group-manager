# 多级标签展开功能优化

## 修改内容

### 1. 支持展开深度选择

新增展开深度选项，支持2级和3级展开：

**3级展开（默认）：**
- 结构：
  - 三级及以上：`A/B/C/D/E` → 组集=A，标签组=B，标签=C、C/D、C/D/E
  - 两级标签：`A/B` → 标签组=A，标签=A/B（不创建组集）
- 适用场景：需要组集来组织大量标签组的情况
- 示例：
  - `前端/框架/React/Hooks/useState` → 组集"前端"，标签组"框架"，标签"React"、"React/Hooks"、"React/Hooks/useState"
  - `前端/React` → 标签组"前端"，标签"前端/React"（不创建组集）

**2级展开：**
- 结构：`A/B/C/D` → 标签组=A，标签=B、B/C、B/C/D（不创建组集）
- 适用场景：标签分类较少，不需要组集的情况
- 示例：`框架/React/Hooks/useState` → 标签组"框架"，标签"React"、"React/Hooks"、"React/Hooks/useState"

### 2. 支持指定第一级标签过滤展开

### 2. 支持指定第一级标签过滤展开

在设置页面的"多级标签适配"部分，新增了一个输入框，允许用户指定要展开的第一级标签。

**使用方法：**
- 输入要展开的第一级标签名称，多个标签用逗号分隔
- 例如：`前端,后端,数据库`
- 留空则展开所有多级标签

**效果：**
- 只有匹配的第一级标签会被展开到组集和标签组
- 其他标签不会被处理

### 3. 收录所有末节点（叶子节点）

**旧逻辑问题：**
- 深度 > 3 的叶子节点会被遗漏
- 例如：`前端/框架/React/Hooks/useState` 中的 `useState` 不会被收录

**新逻辑：**
- 收录所有节点，包括所有深度的末节点
- 所有标签都会被添加到对应的标签组中

### 4. 两级标签特殊处理

在3级展开模式下，对于只有两级的多级标签（如 `前端/React`）：
- 直接以第一级为名生成标签组
- 不创建组集
- 标签组中包含该两级标签

这样可以避免为少量两级标签创建不必要的组集。

### 5. 防命名重复措施

**组集名重复：**
- 如果组集名已存在，自动添加序号：`前端 (1)`、`前端 (2)`
- 确保不会覆盖已有的组集

**标签组名重复：**
- 如果标签组名已存在，自动添加序号：`框架 (1)`、`框架 (2)`
- 确保不会覆盖已有的标签组

这样可以安全地多次运行展开功能，不会破坏已有的配置。

### 6. 末节点显示策略

**显示规则：**

1. **非多级标签**（如 `React`）：总是显示
2. **深度 < minDepth 的多级标签**：总是显示
   - 3级展开：深度 < 3（如 `前端/框架`）
   - 2级展开：深度 < 2（如 `框架`）
3. **深度 ≥ minDepth 的多级标签**：
   - 有子节点的父节点：显示
   - 自动展开的叶子节点：隐藏
   - 手动添加的叶子节点：显示

**各界面的显示：**

**设置页面：**
- 显示所有标签，包括所有深度的末节点
- 用户可以为所有标签设置颜色

**标签选择器和标签组视图：**
- 隐藏深度 ≥ 3 的叶子节点
- 显示所有其他标签（非多级、深度<3、有子节点的父节点）

**悬浮式层级看板：**
- 显示完整的树形结构，包括所有末节点
- 支持点击任意节点插入标签
- 支持颜色显示

**手动添加末节点的处理：**

如果用户手动添加了一个深度 ≥ 3 的末节点标签（如 `前端/框架/React/Hooks/useState`）：
- 该标签会被收录到标签组
- 不会被记录到 `autoExpandedTags`
- 在所有界面（包括标签选择器和标签组视图）都会正常显示
- 可以在设置页面设置颜色
- 用户可以直接在选择器中点击插入

**自动展开的末节点：**
- 会被记录到 `autoExpandedTags`
- 在标签选择器和标签组视图中被隐藏
- 在设置页面显示，可以设置颜色
- 通过悬浮看板访问和插入

**优势：**
- 用户手动添加的标签总是可见，符合用户预期
- 自动展开的末节点被隐藏，保持界面简洁
- 完全向后兼容，旧数据不受影响

## 技术实现

### 新增设置字段
```typescript
interface TagGroupManagerSettings {
  // ...
  expandTagPrefixes: string;  // 指定要展开的第一级标签（逗号分隔）
  expandDepth: number;        // 展开深度（2或3）
}
```

### 数据结构修改
```typescript
interface TagGroup {
  id?: string;
  name: string;
  tags: string[];
  autoExpandedTags?: string[];  // 新增：记录自动展开添加的标签
}
```

**向后兼容性：**
- `autoExpandedTags` 和 `expandDepth` 都是可选字段，旧数据没有这些字段也能正常工作
- 如果字段不存在或为空，所有标签都被视为手动添加，不会被隐藏
- 用户升级插件后，旧数据继续正常工作

### 新增辅助函数
```typescript
function shouldHideTag(tag: string, allTags: string[], autoExpandedTags?: string[], minDepth: number = 3): boolean {
  // 如果不在 autoExpandedTags 中，说明是手动添加的，不隐藏
  if (!autoExpandedTags || !autoExpandedTags.includes(tag)) return false;
  
  // 非多级标签：不隐藏
  if (!tag.includes('/')) return false;
  
  // 深度 < minDepth：不隐藏
  const depth = tag.split('/').length;
  if (depth < minDepth) return false;
  
  // 检查是否有子节点
  const prefix = tag + '/';
  const hasChildren = allTags.some(t => t.startsWith(prefix));
  
  // 没有子节点 = 叶子节点 = 需要隐藏
  return !hasChildren;
}

function getGroupExpandDepth(group: TagGroup, tagGroupSets: TagGroupSet[]): number {
  // 如果标签组在某个组集中，说明是3级展开；否则是2级展开
  if (!group.id) return 2;
  const inSet = tagGroupSets.some(set => set.groupIds.includes(group.id!));
  return inSet ? 3 : 2;
}
```

### 展开逻辑优化
- 支持2级和3级展开深度选择
- 支持前缀过滤
- 收录所有节点（从指定层级开始到最后一层）
- 将自动添加的标签记录到 `autoExpandedTags` 字段

### 渲染逻辑优化
- TagSelectorModal: 根据标签组的展开深度过滤叶子节点
- TagGroupView: 根据标签组的展开深度过滤叶子节点
- 设置页面: 显示所有标签（不过滤）
- 通过检查标签组是否在组集中来判断展开深度（在组集中=3级，不在=2级）

### 向后兼容
- 旧数据没有 `autoExpandedTags` 和 `expandDepth` 字段，所有标签被视为手动添加
- 新的展开操作会创建并使用这些字段
- 用户可以继续手动添加标签，这些标签不会被隐藏

## 使用示例

### 示例1：3级展开（包含两级标签）

**设置：**
```
展开深度: 3级展开
指定展开的第一级标签: 前端,后端
```

**库中的标签：**
```
#前端/React                      (两级)
#前端/Vue                        (两级)
#前端/框架/React                 (三级)
#前端/框架/React/Hooks           (四级)
#前端/框架/React/Hooks/useState  (五级)
#后端/语言/Python                (三级)
#后端/语言/Python/Django         (四级)
#数据库/SQL/MySQL                (三级)
```

**展开结果：**
- 标签组：前端（不创建组集，因为是两级标签）
  - 标签：前端/React
  - 标签：前端/Vue
- 组集：前端
  - 标签组：框架
    - 标签：前端/框架/React
    - 标签：前端/框架/React/Hooks
    - 标签：前端/框架/React/Hooks/useState
- 组集：后端
  - 标签组：语言
    - 标签：后端/语言/Python
    - 标签：后端/语言/Python/Django
- 数据库相关标签不会被展开

**注意：** 如果已存在名为"前端"的标签组或组集，新创建的会自动添加序号，如"前端 (1)"

## 重复展开的行为

### ✅ 不会影响的内容：

1. **标签组顺序**：已存在的标签组不会改变顺序
2. **标签顺序**：已存在的标签不会改变顺序，新标签会被追加到末尾
3. **颜色设置**：颜色存储在独立的 `tagColors` 中，不会被修改
4. **手动添加的标签**：不会被删除或修改
5. **手动删除的标签**：如果用户删除了自动展开的标签，不会被重新添加

### ✅ 会发生的操作：

1. **添加新标签**：如果库中新增了符合条件的多级标签，会被添加到对应的标签组
2. **更新 autoExpandedTags**：新添加的标签会被记录到 `autoExpandedTags`
3. **创建新的标签组/组集**：如果有新的标签分类，会创建对应的标签组或组集

### 智能行为：

**场景1：用户手动删除了自动展开的标签**
```
初始状态：tags = [A, B, C], autoExpandedTags = [A, B, C]
用户删除 B：tags = [A, C], autoExpandedTags = [A, B, C]
再次展开：tags = [A, C]（B 不会被重新添加）
```

**场景2：库中新增了标签**
```
初始状态：tags = [A, B], autoExpandedTags = [A, B]
库中新增 C
再次展开：tags = [A, B, C], autoExpandedTags = [A, B, C]
```

**场景3：用户手动调整了标签顺序**
```
初始状态：tags = [A, B, C]（按字母排序）
用户调整：tags = [C, A, B]（手动排序）
库中新增 D
再次展开：tags = [C, A, B, D]（保持用户的顺序，新标签追加到末尾）
```

### 建议：

- **首次展开后**：可以安全地调整标签组顺序、标签顺序、设置颜色
- **定期重新展开**：当库中新增多级标签时，可以重新展开来添加新标签
- **删除不需要的标签**：删除后不会被重新添加，可以放心删除

## 优势

1. **灵活的展开深度**：支持2级和3级展开，适应不同的标签组织需求
2. **选择性展开**：可以选择性展开特定的标签分类
3. **完整性**：所有末节点都被收录，不会遗漏
4. **可配置性**：末节点可以在设置页面设置颜色
5. **简洁性**：自动展开的末节点在选择器中隐藏，保持界面简洁
6. **灵活性**：手动添加的标签总是显示，符合用户预期
7. **可访问性**：通过悬浮看板可以访问所有节点
8. **向后兼容**：完全兼容旧数据，不会破坏任何现有配置
