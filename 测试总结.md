## 发现的主要问题

### 🔴 高优先级问题

1. **深度>3的叶子节点被忽略**
   - 状态：✅ 已修复
   - 影响：与展开功能不一致
   - 修复：收录所有节点

2. **不支持autoExpandedTags标记**
   - 状态：✅ 已修复
   - 影响：无法区分自动添加和手动添加
   - 修复：添加autoExpandedTags支持

3. **不尊重用户的删除操作**
   - 状态：✅ 已修复
   - 影响：会重新添加被删除的标签
   - 修复：检查wasAutoExpanded状态

### 🟡 中低优先级问题

4. **规则格式验证不足**
   - 状态：✅ 已修复
   - 影响：可能导致解析错误
   - 修复：使用indexOf代替split

5. **性能考虑**
   - 状态：⚠️ 待优化
   - 影响：大型库可能较慢
   - 建议：添加进度提示

## 修复内容

### 修复1：收录所有节点
```typescript
// 旧代码：只收录父节点和深度=3的叶子
if (!isLeafOfCurrentTag || depth === 3) {
    // 添加标签
}

// 新代码：收录所有节点
for (let i = 2; i < tagParts.length; i++) {
    const tagToAdd = tagParts.slice(0, i + 1).join('/');
    // 添加标签
}
```

### 修复2：支持autoExpandedTags
```typescript
// 新增逻辑
if (!group.autoExpandedTags) {
    group.autoExpandedTags = [];
}

const wasAutoExpanded = group.autoExpandedTags.includes(t);
const isCurrentlyInGroup = group.tags.includes(t);

if (!isCurrentlyInGroup && !wasAutoExpanded) {
    // 添加新标签并记录
    group.tags.push(t);
    group.autoExpandedTags.push(t);
}
```

### 修复3：改进规则解析
```typescript
// 旧代码
const parts = rule.split(':');
const pattern = parts[0];
const targetGroupName = parts[1];

// 新代码
const colonIndex = rule.indexOf(':');
const pattern = rule.substring(0, colonIndex).trim();
const targetGroupName = rule.substring(colonIndex + 1).trim();
```

## 向后兼容性

✅ **完全向后兼容**
- 旧数据没有autoExpandedTags字段，会自动创建
- 不会影响已有的手动添加的标签
- 不会破坏现有的规则配置
