## 功能概述

多级标签适配功能旨在帮助用户更好地管理和组织Obsidian中的多级标签(如 `#前端/框架/React`)。该功能包含三个核心模块:

1. **多级标签自动展开**: 将多级标签转化为标签组集和标签组的层级结构
2. **规则匹配自动添加**: 基于关键字或正则表达式自动将标签添加到指定标签组
3. **智能匹配自动添加**: 根据展开深度自动将新标签按层级匹配到对应标签组

## 多级标签自动展开

### 功能说明

自动扫描库中的多级标签,根据配置的展开深度创建对应的标签组集和标签组结构。

### 展开深度

#### 2级展开

**结构**: 标签组 / 标签

**示例**:
```
原始标签: #前端/框架/React, #前端/工具/Webpack, #后端/语言/Python

展开后:
├─ 标签组: 前端
│  ├─ 前端/框架/React
│  ├─ 前端/工具/Webpack
│
└─ 标签组: 后端
   └─ 后端/语言/Python
```

**适用场景**: 按领域/项目分类,适合扁平化的组织结构

#### 3级展开

**结构**: 组集(第一级) / 标签组(第二级) / 标签

**示例**:
```
原始标签: #前端/框架/React, #前端/工具/Webpack, #后端/语言/Python

展开后:
├─ 组集: 前端
│  ├─ 标签组: 框架
│  │  └─ 前端/框架/React
│  │
│  └─ 标签组: 工具
│     └─ 前端/工具/Webpack
│
└─ 组集: 后端
   └─ 标签组: 语言
      └─ 后端/语言/Python
```

**适用场景**: 按功能/类别分类,适合层级化的组织结构

### 配置选项

#### 展开深度
- **2级展开**: 创建标签组,不创建组集
- **3级展开**: 创建组集和标签组

#### 指定展开的第一级标签
- 留空: 展开所有多级标签
- 指定: 只展开指定的第一级标签(逗号分隔)
- 示例: `前端,后端,数据库`

### 使用步骤

1. 在设置中选择展开深度(2级或3级)
2. (可选)指定要展开的第一级标签
3. 点击"立即展开"按钮
4. 系统自动创建标签组集和标签组结构

### 注意事项

- 展开操作会创建新的标签组集和标签组,不会删除现有的
- 已存在的标签组不会被重复创建
- 展开后的标签会被标记为`autoExpandedTags`,用于后续的智能管理

## 规则匹配自动添加

### 功能说明

根据用户定义的规则(关键字或正则表达式),自动将匹配的标签添加到指定的标签组中。

### 规则格式

```
匹配模式:目标标签组名称
```

**示例**:
- `重要:重要事项` - 将包含"重要"的标签添加到"重要事项"组
- `^项目/.*:项目管理` - 将以"项目/"开头的标签添加到"项目管理"组
- `React:前端框架` - 将包含"React"的标签添加到"前端框架"组

### 匹配逻辑

1. **正则表达式优先**: 首先尝试将模式作为正则表达式匹配
2. **字符串包含**: 如果正则表达式失败,则使用字符串包含匹配(不区分大小写)
3. **多级标签处理**: 
   - 对于3级及以上的标签,添加从第三级开始的所有节点
   - 对于2级标签,直接添加完整标签

### 配置选项

#### 添加规则
- **匹配模式**: 关键字或正则表达式
- **目标标签组**: 要添加到的标签组名称(支持自动补全)

#### 启动时自动运行
- 启用后,插件启动时自动执行一次扫描

### 使用步骤

1. 在"自动添加标签至标签组"区域添加规则
2. 输入匹配模式和目标标签组名称
3. 点击"添加规则"按钮
4. 点击"立即刷新"按钮执行扫描
5. (可选)启用"启动时自动运行"

### 示例场景

#### 场景1: 按重要性分类
```
规则: 重要:重要事项
规则: 紧急:紧急事项
规则: 待办:待办事项

效果:
#重要/项目A → 添加到"重要事项"组
#紧急/会议 → 添加到"紧急事项"组
#待办/学习 → 添加到"待办事项"组
```

#### 场景2: 按项目分类
```
规则: ^工作/.*:工作项目
规则: ^个人/.*:个人项目
规则: ^学习/.*:学习项目

效果:
#工作/项目A/需求 → 添加到"工作项目"组
#个人/博客/写作 → 添加到"个人项目"组
#学习/编程/Python → 添加到"学习项目"组
```

#### 场景3: 按技术栈分类
```
规则: React:前端技术
规则: Python:后端技术
规则: Docker:DevOps

效果:
#前端/框架/React → 添加到"前端技术"组
#后端/语言/Python → 添加到"后端技术"组
#DevOps/容器/Docker → 添加到"DevOps"组
```

## 智能匹配自动添加

### 功能说明

配合多级标签展开功能使用,根据展开深度自动将新出现的多级标签按层级匹配到对应的标签组中。

### 核心特性

#### 1. 只处理未被添加的标签
- 只处理还未被添加到任何标签组的标签
- 避免重复添加和干扰用户的手动管理
- 保护用户的手动操作

#### 2. 根据展开深度自动选择匹配模式
- **2级展开** → **level1模式**(按第一级匹配)
- **3级展开** → **level2模式**(按第二级匹配)
- 切换展开深度时自动更新匹配模式

#### 3. 找不到就不添加
- 不是优先级的概念,而是**二选一**
- level1模式: 只按第一级匹配,找不到就不添加
- level2模式: 只按第二级匹配,找不到就不添加

### 匹配模式详解

#### level1模式(适配2级展开)

**工作原理**: 只按多级标签的第一级进行匹配

**示例**:
```
标签: #前端/框架/React
匹配: 查找名为"前端"的标签组

结果:
✅ 如果存在"前端"组 → 添加到"前端"组
❌ 如果不存在"前端"组 → 不添加

不会尝试:
❌ 不会尝试匹配"框架"组(即使存在)
❌ 不会尝试匹配其他任何组
```

**适用场景**:
- 按领域分类: 前端、后端、DevOps
- 按项目分类: 工作、个人、学习
- 扁平化的组织结构

#### level2模式(适配3级展开)

**工作原理**: 只按多级标签的第二级进行匹配

**示例**:
```
标签: #前端/框架/React
匹配: 查找名为"框架"的标签组

结果:
✅ 如果存在"框架"组 → 添加到"框架"组
❌ 如果不存在"框架"组 → 不添加

不会尝试:
❌ 不会尝试匹配"前端"组(即使存在)
❌ 不会尝试匹配其他任何组
```

**适用场景**:
- 按功能分类: 框架、语言、工具
- 按类型分类: 方法论、案例、工具
- 层级化的组织结构

### 与规则匹配的关系

#### 执行顺序

```
1. 检查标签是否已在任何标签组中
   ↓ 如果未被添加
2. 尝试智能匹配
   ↓ 如果智能匹配成功
3. 跳过规则匹配
   ↓ 如果智能匹配失败
4. 继续执行规则匹配
```

#### 互补使用

**智能匹配**:
- 自动按层级匹配
- 适合有规律的多级标签结构
- 配合展开功能使用
- 优先执行

**规则匹配**:
- 基于关键字或正则表达式
- 适合灵活的自定义规则
- 可以处理智能匹配无法处理的标签
- 在智能匹配之后执行

**示例**:
```
智能匹配(level2): #前端/框架/Svelte → 添加到"框架"组
规则匹配: #重要 → 添加到"重要事项"组(通过规则: 重要:重要事项)
```

### 配置选项

#### 启用智能匹配
- 开关控制功能的启用/禁用

#### 当前匹配模式(只读)
- 显示当前使用的匹配模式
- 根据展开深度自动设置
- 包含示例说明

### 使用步骤

1. 先使用"多级标签自动展开"功能创建标签组结构
2. 在"多级标签智能匹配"区域启用功能
3. 系统自动根据展开深度设置匹配模式
4. 添加新的多级标签时,自动匹配到对应标签组
5. (可选)点击"立即刷新"按钮手动执行扫描

## 完整工作流程

### 初始设置流程

```
步骤1: 选择展开深度
├─ 2级展开: 按领域/项目分类
└─ 3级展开: 按功能/类别分类

步骤2: 执行多级标签展开
├─ 点击"立即展开"按钮
├─ 系统创建标签组集和标签组
└─ 现有标签被自动归类

步骤3: 启用智能匹配
├─ 开启"启用智能匹配"开关
├─ 系统自动设置匹配模式
└─ 新标签将自动归类

步骤4: (可选)添加规则匹配
├─ 添加自定义匹配规则
├─ 处理特殊标签
└─ 补充智能匹配的不足
```

### 日常使用流程

```
添加新标签
↓
智能匹配检查
├─ 标签已在组中? → 跳过
├─ 找到匹配的组? → 添加到该组
└─ 未找到匹配? → 继续规则匹配
    ↓
    规则匹配检查
    ├─ 匹配到规则? → 添加到目标组
    └─ 未匹配? → 保持未分类状态
```

## 最佳实践

### 1. 规划标签层级结构

**2级展开适合**:
- 按领域分类: 前端、后端、DevOps
- 按项目分类: 工作、个人、学习
- 扁平化的组织结构
- 标签数量较少的场景

**3级展开适合**:
- 按功能分类: 框架、语言、工具
- 按类型分类: 方法论、案例、工具
- 层级化的组织结构
- 标签数量较多的场景

### 2. 合理使用三种功能

**多级标签展开**:
- 用于初始化标签组结构
- 一次性操作,不需要频繁执行
- 适合已有大量多级标签的情况

**智能匹配**:
- 用于日常的自动归类
- 配合展开功能使用
- 适合有规律的多级标签

**规则匹配**:
- 用于特殊标签的处理
- 补充智能匹配的不足
- 适合灵活的自定义需求

### 3. 保持标签组名简洁

- 标签组名应该简洁明确
- 便于智能匹配识别
- 避免使用特殊字符

### 4. 定期检查未匹配的标签

- 使用规则匹配处理特殊标签
- 或手动添加到合适的标签组
- 保持标签组织的完整性

### 5. 灵活调整展开深度

- 根据实际使用情况调整
- 2级和3级各有优势
- 可以随时切换并重新展开

### 6. 结合其他功能

- 使用标签颜色管理增强视觉效果
- 使用标签组集组织复杂结构
- 使用浮动标签选择器快速插入

## 常见问题

### Q1: 智能匹配和规则匹配有什么区别?

**智能匹配**:
- 自动按层级匹配
- 配合展开功能使用
- 只处理未分类的标签
- 找不到就不添加

**规则匹配**:
- 基于关键字或正则表达式
- 可独立使用
- 可以处理任何标签
- 更灵活但需要手动配置

### Q2: 为什么我的标签没有被自动添加?

可能的原因:
1. 智能匹配未启用
2. 标签已经在其他标签组中
3. 没有找到匹配的标签组
4. 匹配模式与展开深度不匹配

解决方法:
1. 检查智能匹配是否启用
2. 检查标签是否已在其他组中
3. 确认存在名称匹配的标签组
4. 重新加载插件或重启Obsidian

### Q3: 可以同时使用2级和3级展开吗?

不建议。展开深度决定了标签组的结构,应该保持一致。如果需要切换:
1. 修改展开深度设置
2. 重新执行展开操作
3. 系统会自动调整智能匹配模式

### Q4: 智能匹配会重复添加标签吗?

不会。智能匹配只处理未被添加到任何标签组的标签,避免重复添加。

### Q5: 如何处理智能匹配无法处理的标签?

有三种方法:
1. 使用规则匹配添加自定义规则
2. 手动添加到合适的标签组
3. 创建对应的标签组,让智能匹配自动处理

## 技术细节

### 标签处理逻辑

```typescript
// 伪代码示例
for (const tag of allTags) {
    // 1. 检查是否已在任何标签组中
    if (isTagInAnyGroup(tag)) {
        continue; // 跳过已分类的标签
    }
    
    // 2. 尝试智能匹配
    if (smartMatchEnabled) {
        const matchedGroup = smartMatch(tag, matchMode);
        if (matchedGroup) {
            addToGroup(tag, matchedGroup);
            continue; // 成功匹配,跳过规则匹配
        }
    }
    
    // 3. 尝试规则匹配
    for (const rule of rules) {
        if (matchRule(tag, rule)) {
            addToGroup(tag, rule.targetGroup);
            break; // 匹配成功,停止尝试其他规则
        }
    }
}
```

### 自动标记机制

```typescript
// 标签组结构
interface TagGroup {
    id: string;
    name: string;
    tags: string[];              // 所有标签
    autoExpandedTags: string[];  // 自动添加的标签
}

// 用途:
// 1. 区分手动添加和自动添加的标签
// 2. 尊重用户的删除操作(不重新添加被删除的标签)
// 3. 支持智能管理和清理
```
