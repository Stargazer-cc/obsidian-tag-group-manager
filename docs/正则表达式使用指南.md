## 什么是正则表达式

正则表达式(Regular Expression,简称Regex)是一种强大的文本模式匹配工具,可以用简洁的语法描述复杂的匹配规则。

### 为什么使用正则表达式?

在Tag Group Manager中,正则表达式可以帮助你:
- 批量匹配多个相似的标签
- 灵活定义标签匹配规则
- 自动化标签管理流程
- 处理复杂的标签命名模式

---

## 在插件中的应用场景

### 1. 批量颜色设置

在"批量颜色操作"区域,使用正则表达式匹配多个标签并统一设置颜色。

**示例**:
```
正则表达式: ^电.*
效果: 匹配所有以"电"开头的标签(电影、电视、电脑等)
```

### 2. 规则匹配自动添加

在"自动添加标签至标签组"功能中,使用正则表达式定义匹配规则。

**示例**:
```
规则: ^项目/.*:项目管理
效果: 将所有以"项目/"开头的标签添加到"项目管理"组
```

### 3. 标签筛选

在各种标签选择和筛选场景中,使用正则表达式快速定位目标标签。

## 基础语法

### 字符匹配

| 语法 | 说明 | 示例 | 匹配结果 |
|------|------|------|----------|
| `.` | 匹配任意单个字符 | `电.` | 电影、电视、电脑 |
| `\d` | 匹配任意数字(0-9) | `\d+` | 123、2023、42 |
| `\w` | 匹配字母、数字或下划线 | `\w+` | abc、test_123 |
| `\s` | 匹配空白字符(空格、制表符等) | `\s+` | 空格、多个空格 |
| `[abc]` | 匹配方括号中的任意一个字符 | `[电视音]乐` | 电乐、视乐、音乐 |
| `[^abc]` | 匹配不在方括号中的任意字符 | `[^电]影` | 音影、视影(不匹配"电影") |
| `[a-z]` | 匹配a到z之间的任意字符 | `[a-z]+` | abc、test |
| `[0-9]` | 匹配0到9之间的任意数字 | `[0-9]+` | 123、2023 |

### 数量限定符

| 语法 | 说明 | 示例 | 匹配结果 |
|------|------|------|----------|
| `*` | 匹配前面的字符0次或多次 | `科.*` | 科、科幻、科技、科学幻想 |
| `+` | 匹配前面的字符1次或多次 | `编程+` | 编程、编程程 |
| `?` | 匹配前面的字符0次或1次 | `读书?` | 读、读书 |
| `{n}` | 精确匹配n次 | `\d{4}` | 2023、1999 |
| `{n,}` | 至少匹配n次 | `\d{2,}` | 23、123、2023 |
| `{n,m}` | 匹配n到m次 | `\d{2,4}` | 23、123、2023 |

### 位置锚点

| 语法 | 说明 | 示例 | 匹配结果 |
|------|------|------|----------|
| `^` | 匹配字符串开头 | `^电影` | 电影(不匹配"看电影") |
| `$` | 匹配字符串结尾 | `笔记$` | 读书笔记(不匹配"笔记本") |
| `\b` | 匹配单词边界 | `\b电影\b` | "电影"(不匹配"电影院") |

### 逻辑运算

| 语法 | 说明 | 示例 | 匹配结果 |
|------|------|------|----------|
| `\|` | 或运算符 | `电影\|电视` | 电影 或 电视 |
| `()` | 分组 | `(电影\|电视)剧` | 电影剧、电视剧 |

### 特殊字符转义

如果要匹配特殊字符本身,需要使用反斜杠 `\` 转义:

| 特殊字符 | 转义写法 | 示例 |
|---------|---------|------|
| `.` | `\.` | `文件\.txt` 匹配 "文件.txt" |
| `*` | `\*` | `重要\*` 匹配 "重要*" |
| `+` | `\+` | `C\+\+` 匹配 "C++" |
| `?` | `\?` | `什么\?` 匹配 "什么?" |
| `^` | `\^` | `\^符号` 匹配 "^符号" |
| `$` | `\$` | `\$100` 匹配 "$100" |
| `\` | `\\` | `路径\\文件` 匹配 "路径\文件" |
| `\|` | `\\\|` | `A\\\|B` 匹配 "A\|B" |
| `()` | `\(\)` | `\(注释\)` 匹配 "(注释)" |
| `[]` | `\[\]` | `\[标签\]` 匹配 "[标签]" |
| `/` | `\/` | `路径\/文件` 匹配 "路径/文件" |

## 实用示例

### 基础匹配

#### 示例1: 匹配所有以"电"开头的标签

```
正则表达式: ^电.*
匹配结果: 电影、电视、电脑、电子书
不匹配: 看电影、家电
```

#### 示例2: 匹配所有包含"笔记"的标签

```
正则表达式: .*笔记.*
匹配结果: 读书笔记、工作笔记、笔记本、学习笔记整理
不匹配: 记录、备忘
```

#### 示例3: 精确匹配"电影"或"电视"

```
正则表达式: ^(电影|电视)$
匹配结果: 电影、电视
不匹配: 电影院、电视剧、看电影
```

### 数字和日期

#### 示例4: 匹配所有纯数字标签

```
正则表达式: ^\d+$
匹配结果: 2023、001、42、12345
不匹配: 2023年、编号001、42号
```

#### 示例5: 匹配包含年份的标签

```
正则表达式: .*20\d{2}.*
匹配结果: 2020年、电影2023、2021-01-01
不匹配: 1999、2030(因为是20xx格式)
```

#### 示例6: 匹配日期格式标签

```
正则表达式: ^\d{4}-\d{2}-\d{2}$
匹配结果: 2023-12-25、2024-01-01
不匹配: 2023/12/25、20231225
```

### 多级标签

#### 示例7: 匹配所有多级标签

```
正则表达式: .*/.*
匹配结果: 工作/项目、学习/编程、生活/健康
不匹配: 工作、学习、生活(单级标签)
```

#### 示例8: 匹配特定前缀的多级标签

```
正则表达式: ^工作/.*
匹配结果: 工作/项目、工作/会议、工作/文档
不匹配: 个人/工作、学习/工作技能
```

#### 示例9: 匹配三级及以上的标签

```
正则表达式: .*/.*/.+
匹配结果: 前端/框架/React、后端/语言/Python
不匹配: 前端/框架、后端(少于三级)
```

### 分类匹配

#### 示例10: 匹配多个特定前缀

```
正则表达式: ^(工作|学习|生活)
匹配结果: 工作、工作/项目、学习笔记、生活记录
不匹配: 娱乐、其他
```

#### 示例11: 匹配特定后缀

```
正则表达式: .*(笔记|记录|总结)$
匹配结果: 读书笔记、工作记录、学习总结
不匹配: 笔记本、记录片
```

#### 示例12: 匹配包含特定关键词的标签

```
正则表达式: .*(重要|紧急|待办).*
匹配结果: 重要事项、紧急任务、待办清单、重要且紧急
不匹配: 普通任务、已完成
```

### 复杂模式

#### 示例13: 匹配编程语言标签

```
正则表达式: ^(JavaScript|Python|Java|C\+\+|Go|Rust)$
匹配结果: JavaScript、Python、Java、C++、Go、Rust
不匹配: TypeScript、C#、Ruby
```

#### 示例14: 匹配项目编号格式

```
正则表达式: ^PRJ-\d{4}$
匹配结果: PRJ-0001、PRJ-2023
不匹配: PRJ-01、项目-0001
```

#### 示例15: 匹配版本号格式

```
正则表达式: ^v\d+\.\d+\.\d+$
匹配结果: v1.0.0、v2.3.5、v10.20.30
不匹配: v1.0、1.0.0、version1.0.0
```

## 高级技巧

### 1. 非贪婪匹配

默认情况下,`*`、`+`、`?` 是贪婪的,会尽可能多地匹配。添加 `?` 可以变为非贪婪匹配。

```
贪婪匹配: .*标签
文本: "这是标签1和标签2"
匹配: "这是标签1和标签" (匹配到最后一个"标签")

非贪婪匹配: .*?标签
文本: "这是标签1和标签2"
匹配: "这是标签" (匹配到第一个"标签")
```

### 2. 前瞻和后顾

#### 正向前瞻 `(?=...)`

匹配后面跟着特定模式的位置。

```
正则表达式: 工作(?=/项目)
匹配: "工作/项目" 中的 "工作"
不匹配: "工作/会议" 中的 "工作"
```

#### 负向前瞻 `(?!...)`

匹配后面不跟着特定模式的位置。

```
正则表达式: 工作(?!/项目)
匹配: "工作/会议" 中的 "工作"
不匹配: "工作/项目" 中的 "工作"
```

### 3. 分组捕获

使用括号 `()` 可以创建捕获组,提取匹配的部分。

```
正则表达式: ^(\w+)/(\w+)$
文本: "前端/React"
捕获组1: "前端"
捕获组2: "React"
```

### 4. 命名捕获组

使用 `(?<name>...)` 可以给捕获组命名。

```
正则表达式: ^(?<category>\w+)/(?<item>\w+)$
文本: "前端/React"
捕获组category: "前端"
捕获组item: "React"
```

## 常见问题

### Q1: 为什么我的正则表达式不工作?

**可能原因**:
1. 特殊字符没有转义
2. 语法错误
3. 大小写不匹配

**解决方法**:
1. 检查是否需要转义特殊字符(如 `.`、`*`、`+` 等)
2. 使用在线工具测试正则表达式
3. 注意正则表达式默认区分大小写

### Q2: 如何匹配包含斜杠的标签?

斜杠 `/` 在多级标签中很常见,需要转义:

```
错误: ^工作/项目$
正确: ^工作\/项目$
```

### Q3: 如何忽略大小写?

在插件中,正则表达式默认不区分大小写。如果需要区分大小写,请在规则中明确指定。

### Q4: 如何匹配中文字符?

中文字符可以直接使用,不需要特殊处理:

```
正则表达式: ^中文.*
匹配结果: 中文标签、中文笔记
```

### Q5: 如何测试我的正则表达式?

建议使用在线工具测试:
- [Regex101](https://regex101.com/) - 功能强大,有详细解释
- [RegExr](https://regexr.com/) - 界面友好,实时测试
- [RegexTester](https://www.regextester.com/) - 简单直观

## 实战练习

### 练习1: 标签分类

**需求**: 将标签按类型分类
- 工作相关: 以"工作"、"项目"、"会议"开头
- 学习相关: 以"学习"、"课程"、"笔记"开头
- 生活相关: 以"生活"、"健康"、"娱乐"开头

**答案**:
```
工作相关: ^(工作|项目|会议)
学习相关: ^(学习|课程|笔记)
生活相关: ^(生活|健康|娱乐)
```

### 练习2: 日期标签

**需求**: 匹配不同格式的日期标签
- YYYY-MM-DD 格式
- YYYY/MM/DD 格式
- YYYYMMDD 格式

**答案**:
```
YYYY-MM-DD: ^\d{4}-\d{2}-\d{2}$
YYYY/MM/DD: ^\d{4}\/\d{2}\/\d{2}$
YYYYMMDD: ^\d{8}$
所有格式: ^\d{4}[-/]?\d{2}[-/]?\d{2}$
```

### 练习3: 多级标签筛选

**需求**: 匹配特定结构的多级标签
- 二级标签: 类别/项目
- 三级标签: 类别/子类/项目
- 排除单级标签

**答案**:
```
二级标签: ^[^/]+/[^/]+$
三级标签: ^[^/]+/[^/]+/[^/]+$
至少二级: .*/.*
```
